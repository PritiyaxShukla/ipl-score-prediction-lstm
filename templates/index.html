Model Overview & Stacked LSTM Architecture

The IPL Score Predictor utilizes a deep learning **Stacked LSTM (Long Short-Term Memory)** network trained on ball-by-ball IPL match data spanning 2008 to 2024.

Unlike static statistical run rate models, this architecture reads the progression of completed overs as a sequence of state vectors, learning complex temporal dependencies like batting acceleration in the death overs, bowling pressure, and wicket loss patterns relative to stadium size and matchups.

Sequence Inputs & Features

For every completed over (up to 20), the model is fed a sequence containing 7 normalized features in this exact order:

  • Cumulative Runs: Runs scored up to the current over snapshot.
  • Cumulative Wickets: Wickets lost up to the current over snapshot.
  • Current Run Rate: Runs divided by overs completed (smoothed to prevent division by zero).
  • Current Over: Timestep index mapping match progression (1 to 20).
  • Batting Team ID: Categorical integer label (fitted on team encoder).
  • Bowling Team ID: Categorical integer label (fitted on team encoder).
  • Venue ID: Categorical integer label (fitted on venue encoder).

Network Architecture

Input Layer 20 Timesteps x 7 Features
↓
LSTM Layer 1 (128 units) return_sequences=True
↓
Dropout (0.3) 30% units deactivated
↓
LSTM Layer 2 (64 units) return_sequences=False
↓
Dropout (0.3) 30% units deactivated
↓
Dense Layer 3 (32 units) ReLU Activation
↓
Dense Output Layer (1 unit) Linear Activation (Scaled Final Score)

Training Configuration & Performance

Metric Test Dataset Performance
Mean Absolute Error (MAE) 1.98 runs
Root Mean Squared Error (RMSE) 2.64 runs
R-squared (R²) Coefficient 0.993
Max Epochs / Early Stopping 150 epochs (Stopped at epoch 110, patience 12)
Best Epoch Model Restore Restored weights from Epoch 98

Note on High R²: The high coefficient is due to the model evaluating sequence snapshots where late-innings overs contain cumulative run details very close to the target final score. Feature scaling is normalized via a fitted MinMaxScaler and final targets are normalized with a 300.0 scale factor for training stability.
